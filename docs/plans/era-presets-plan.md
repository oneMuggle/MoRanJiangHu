# 时代适配天赋/背景/气运 计划

## 需求

由于现在有多个时代的设定，开局设置中的天赋、背景、气运等都需要增加适配时代的条目，后续将根据时代筛选这些条目。

## 现状

- **天赋**：`types.ts` 定义 `天赋结构`（名称/描述/效果/适用性别/nsfw），`data/presets.ts` 定义 ~55 条预设天赋
- **背景**：`types.ts` 定义 `背景结构`（名称/描述/效果/适用性别/nsfw），`data/presets.ts` 定义 ~60 条预设背景
- **气运**：`data/qiyun/index.ts` 定义 `气运数据`（名称/类别/描述/效果/稀有度/nsfw等级/能力类型/适用境界），~250 条目
- **当前过滤**：仅支持性别（适用性别）+ NSFW 开关过滤，无时代过滤
- **时代体系**：`models/system.ts` 定义 `全部时代配置`（~25 个时代，含旧ID和新ID），`models/eraTheme/` 有树状时代结构

## 方案

### Step 1：扩展类型定义

在以下接口中新增 `时代适配?: string[]` 字段（时代ID列表，为空表示全时代通用）：

| 接口 | 文件 | 行号 |
|---|---|---|
| `天赋结构` | `types.ts` | 88-96 |
| `背景结构` | `types.ts` | 98-106 |
| `气运数据` | `data/qiyun/index.ts` | 20-35 |

```typescript
// types.ts - 天赋结构 / 背景结构 新增
时代适配?: string[]; // 适配的时代ID列表，未设置则全时代可用

// data/qiyun/index.ts - 气运数据 新增
时代适配?: string[]; // 适配的时代ID列表，未设置则全时代可用
```

### Step 2：为现有预设打标签

**原则**：不设置 `时代适配` 的条目默认全时代可用（向后兼容），仅对明显特定时代的内容打标。

| 数据集 | 文件 | 打标策略 |
|---|---|---|
| 天赋 | `data/presets.ts` | 科幻/赛博相关天赋（如"科学修仙"等）打标对应时代；通用天赋不标 |
| 背景 | `data/presets.ts` | 现代/赛博/特定时代背景打标；古代武侠通用背景不标 |
| 气运 | `data/qiyun/index.ts` | 科幻类气运（"科学修仙"等）打标对应时代；通用气运不标 |

> 注意：大部分条目不需要打标。不设置 `时代适配` 的条目在所有时代都可见。

### Step 3：在 wizard 状态中增加时代过滤

修改 `components/features/NewGame/useNewGameWizardState.ts` 中的三个 useMemo：

1. **全部背景选项**（~206行）：在现有性别+NSFW过滤基础上，增加时代匹配
2. **全部天赋选项**（~214行）：同上
3. **全部气运选项**（~222行）：同上

过滤逻辑：

```typescript
const 当前时代 = worldConfig.时代配置ID;
const 匹配时代 = (item: { 时代适配?: string[] }) =>
    !item.时代适配 || item.时代适配.length === 0 || item.时代适配.includes(当前时代);
```

将此条件 AND 到现有的过滤条件中。

### Step 4（可选）：UI 提示

在天赋/背景/气运选择界面中，对设置了时代适配的条目显示时代标签（如小 badge），帮助用户理解为什么某些条目在特定时代不可见。

## 涉及文件

| 文件 | 改动类型 |
|---|---|
| `types.ts` | 接口扩展（+1字段×2接口） |
| `data/qiyun/index.ts` | 接口扩展（+1字段）+ 部分数据打标 |
| `data/presets.ts` | 部分天赋/背景打标 |
| `components/features/NewGame/useNewGameWizardState.ts` | 过滤逻辑增强 |

## 风险评估

- **低风险**：`时代适配` 是可选字段，不影响已有数据
- **兼容性**：未设置该字段的条目默认全时代可见，不会丢失任何选项
- **性能**：过滤逻辑是简单的 array.includes 操作，数据量小（<300条），无性能问题
