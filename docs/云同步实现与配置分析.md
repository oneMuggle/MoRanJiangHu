# 云同步实现与配置分析

## 一、架构概览

该项目使用 **GitHub 私有仓库 + Release 附件** 作为云同步后端，存储位置是 GitHub Release 的分卷上传附件。

技术栈：
- **压缩库**：fflate（ZIP 打包/解包）
- **存储后端**：GitHub 私有仓库 + Release 附件
- **认证**：GitHub OAuth（或 Personal Access Token）
- **代理**：Cloudflare Pages Functions（可选）

## 三、同步原理

### 3.1 数据打包

使用 `fflate` 库将以下内容打包为 ZIP：

```
WuXia_Save_Data.zip/
├── manifest.json           # 清单元数据
├── saves/
│   ├── index.json        # 存档索引
│   └── manual_*.json    # 手动存档
│   └── auto_*.json      # 自动存档
├── settings/
│   ├── index.json      # 设置索引
│   └── *.json          # 各类设置
└── assets/bin/
    ├── index.json      # 图片资源索引
    └── *.jpg/png     # 图片二进制
```

### 3.2 分卷上传

大文件自动切分为 **8MB 分卷**，解决 GitHub 单个附件 2GB 限制：

```typescript
const 云同步分卷大小 = 8 * 1024 * 1024; // 8MB
```

### 3.3 存储位置

- **仓库**：自动在用户 GitHub 账号下创建私有仓库（默认名 `wuxia-cloud-save`）
- **Release 标签**：`wuxia-cloud-sync`
- **附件**：分卷 + 清单文件（`.parts.json`）

### 3.4 核心 API

```typescript
// services/githubSync.ts
await uploadToCloud(token, onProgress)    // 上传
await downloadFromCloud(token, onProgress) // 下载恢复
await fetchSyncMetaData(token)           // 查询远端状态
await restoreSyncData(zipBytes)           // 恢复数据
```

1. GitHub → Settings → Developer settings
2. Personal access tokens → Tokens (classic)

4. 将 token 填入应用提示框

## 八、开发环境问题排查

### 未配置 VITE_GITHUB_CLIENT_ID

**症状**：点击云存档登录时提示"未配置 VITE_GITHUB_CLIENT_ID 环境变量"

**原因**：`hooks/useGitHubOAuth.ts` 从 Vite 环境变量读取配置

**解决**：在项目根目录创建 `.env.local`

```bash
echo 'VITE_GITHUB_CLIENT_ID=your_client_id' > .env.local
npm run dev
```

### 替代方案

如无 OAuth App，可使用 **Personal Access Token** 直接操作：

1. 获取 Token：GitHub → Settings → Developer settings → Personal access tokens
2. 首次上传时选择"直接使用 Token"模式
3. 输入 Token 后即可使用

// 后端：交换 token

    method: 'POST',
    body: JSON.stringify({
        client_id: GITHUB_CLIENT_ID,
        client_secret: GITHUB_CLIENT_SECRET,

    })
})

## 十二、OAuth 配置与用户登录授权的区别

### 12.1 两个不同的概念

| 概念 | 说明 | 谁做 |
|------|------|-----|
| **OAuth 配置**（`VITE_GITHUB_CLIENT_ID`） | 开发者在 GitHub 注册的 OAuth App 身份凭证 | 开发者（一次性） |
| **用户登录授权** | 每次用户点击登录时，GitHub 询问是否允许访问 | 用户（每次） |

### 12.2 流程对比

```
OAuth App 配置（开发者做一次）
┌─────────────────────────────────────┐
│  开发者去 GitHub 创建 OAuth App          │
│  → 获得 Client ID + Client Secret     │
└─────────────────────────────────────┘
                    ▼
用户登录授权（每个用户每次登录）
┌─────────────────────────────────────┐
│  1. 用户点击"云存档登录"             │
│  2. 跳转到 GitHub 授权页面         │
│  3. 用户点击"Authorize xxx"       │
│  4. GitHub 返回 code              │
│  5. 用 code 换 access_token      │
│  6. Token 存入 localStorage       │
└─────────────────────────────────────┘
```

### 12.3 两者分工

| 配置 | 用途 |
|------|------|
| `VITE_GITHUB_CLIENT_ID` | **登录授权** - 点击"云存档登录"后跳转到 GitHub OAuth 页面 |
| 仓库名 | **数据存储** - 首次上传时输入，指定私有仓库名 |

## 十三、当前设计缺陷

### 13.1 问题

当前代码在 `hooks/useGitHubOAuth.ts:60-67`：

```typescript
const login = () => {
    const clientId = (import.meta as any).env.VITE_GITHUB_CLIENT_ID;
    if (!clientId) {
        alert('未配置 VITE_GITHUB_CLIENT_ID 环境变量');
        return;  // ← 直接退出，没有替代方案
    }
    window.location.href = `https://github.com/login/oauth/authorize?client_id=${clientId}&scope=repo`;
};
```

当没有配置 `VITE_GITHUB_CLIENT_ID` 时，代码直接退出，没有提供替代登录方式。

### 13.2 死胡同

```
如果无 OAuth 配置
    → alert 退出
    → 无法登录
```

应该改为：

```
如果无 OAuth 配置
    → 让用户手动输入 Token（PAT）
```

### 13.3 临时解决方案

在浏览器控制台手动设置 Token：

```javascript
// 在浏览器 console 执行
localStorage.setItem('github_sync_token', 'YOUR_GITHUB_TOKEN')
location.reload()
```

获取 Token：
1. GitHub → Settings → Developer settings
2. Personal access tokens → Tokens (classic)
3. Generate new token（勾选 `repo` 权限）

## 十四、多用户隔离机制

每个用户的云同步是**独立的**：

1. **每个用户独立仓库**：首次使用时会弹出 `window.prompt` 让用户输入仓库名
2. **绑定到用户账号**：仓库创建在当前 GitHub 账号下
3. **本地存储**：仓库名 + Token 存储在 `localStorage` 中

```typescript
// services/githubSync.ts:313-317
const 获取仓库配置 = async (token: string): Promise<RepoConfig> => {
    const user = await 获取当前用户(token);  // 获取当前 GitHub 用户
    const repo = 获取或询问仓库名();            // 用户输入仓库名
    return { owner: user.login, repo };    // owner=用户名, repo=仓库名
};
```

不同用户的数据互不影响，因为存储在各自 GitHub 账号下的私有仓库中。
