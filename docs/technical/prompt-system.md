# 提示词系统

> 扩展自 `prompts/AGENTS.md` | **日期:** 2026-04-28

## 概述

项目使用分层提示词工程体系，~160 个文件，6 层目录结构。

## 目录结构

```
prompts/
├── core/       # 核心规则和思维链（19 文件）
├── runtime/    # 运行时链路（38 文件）
├── writing/    # 写作风格约束（5 文件）
├── stats/      # 数值系统规则（13 文件）
├── difficulty/ # 难度判定规则
├── shared/     # 跨链路共享默认值
└── index.ts    # 导出 默认提示词 数组
```

## 层级详情

### core/ — 核心规则

| 文件 | 用途 |
|------|------|
| `rules.ts` | 核心游戏规则 |
| `format.ts` | 输出格式定义 |
| `cot.ts` | 主流程思维链 |
| `cotCombat.ts` | 战斗思维链 |
| `cotJudge.ts` | 判定思维链 |
| `memory.ts` | 记忆法则 |
| `realm.ts` | 境界体系 |

### runtime/ — 运行时链路

- 开局剧情生成
- 世界生成和演变
- 变量生成和校准
- 规划分析
- 世界演变工作流
- NSFW 内容规则
- 亲密系统规则
- 气运（运气）系统
- 图片提取提示词

### writing/ — 写作风格

- 视角约束
- 散文风格指南
- 时代感知写作风格（关联 `models/eraTheme.ts`）

### stats/ — 统计规则

- 经验和升级
- 战斗计算
- 角色属性
- 物品掉落率
- 世界统计

### difficulty/ — 难度

- 判定和难度缩放规则

### shared/ — 跨链路共享

- 在多个提示词层复用的通用默认值和辅助函数

## 如何添加新提示词

1. 确定提示词属于哪一层
2. 在对应目录下创建新文件
3. 如需加入默认提示词数组，从 `prompts/index.ts` 导出
4. 遵循现有的文件命名和导出约定

## 核心原则

- **关注点分离**：每层处理 AI 行为的特定方面
- **可组合性**：运行时提示词从 core + writing + stats 组装
- **共享默认**：通用模式提取到 `shared/`
